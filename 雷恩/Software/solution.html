网络监控管理 SolarWinds 网络监控解决方案 一、现状

网络对于业务越来越重要，如何更好地对网络进行监控、维护，如何减少网络故障的发生， 保证业务的正常开展，是现实的问题。网络性能管理不仅仅是测量服务水准的目标，它还应该包括对网速降低或者通信中断的及时发现、基于事实的容量规划、对网络资源有效的使用，以及全面的故障排除。目前可能存在的困惑：

1.对于庞大的网络环境，出现故障不易查找、诊断和修复：没有准确的数据可以进行科学化的故障分析，只能是低效率盲目地猜测；

2.没有预防性管理：没有预防性能力，在网络问题出现前没有任何预警；

3.无效的应用监控：没有对网络中的应用进行有效的监控，当应用系统性能下降时，不能及时判断出是网络问题，还是应用系统本身问题，无法保证业务的连续运行；

4.网络基础设施的性能利用率低：不能充分利用现有的带宽和设备，无法发现带宽瓶颈；

5.不能实现有效的网络规划：不能为网络增长提供基于事实的依据；

6.网络的可视性差，想一览全局状态很困难；

7.手动的对众多的网络设备（如交换机，路由器，防火墙）和服务器（SMTP，WWW，DNS， SQL）进行监控，浪费管理员的精力和时间，而且不能及时发现问题所在；

8.当发生故障的时候，不能及时得到有效的警报信息；

9.没有一个简观明了的图表功能，说明网络或节点的相关资源使用情况。

完善的网络性能管理解决方案，应该满足对网络性能日益提高的需求，提供最佳性能管理和报告工具，帮助网络管理人员从被动管理模式转换至主动识别流量变化模式，并针对不断提高的网络需求进行规划。网络带宽和网络服务级别的监测，网络仿真则可在所建议的网络变化实施之前对其加以测试。网络中所有的存储与备份及虚拟化系统都应该有一体化的监控平台。网络中所有的日志系统都能够被自动收集与分析。同时，方案应具有可扩展性，要具备随网络发展而灵活扩充的能力。这些都可以通过我们建议的使用SolarWinds的IT综合环境监控解决方案得到满足。

二、解决方案概述

SolarWinds是一个支持分布式部署的综合IT环境监控解决方案，用于网络性能监测、流量分析、链路监控、配置管理、虚拟化监控、日志监控、存储与备份系统监控等IT的多个方面。该系统根据管理员配置的任务模型主动、智能和协作地监控网络，从网络节点收集数据，并分析出其潜在的错误。Solarwinds是专门为各种复杂的网络环境开发的网络管理、网络监控，应用监控，虚拟化监控，存储与备份系统监控，日志监控的综合IT环境解决方案。能够满足电信运营商、大中院校、企事业单位对网络管理和专业咨询的不同需求。

SolarWinds解决方案具有以下优势：

1.快捷安装以及简便使用：Solarwinds所有模块可在数个小时内部署完毕并立即投入生产使用，维护升级也十分简便，极大缩短系统熟悉周期，节约了维护成本；

2.监控全面：支持国内外各厂商的各类型主流设备，包括网络设备、安全设备、负载均衡设备、VMware、存储产品、备份软件等；

3.涵盖网管领域的各项功能：包括性能监控、流量分析、链路监控、IP地址监控、日志收集分析及配置管理等方面；

4.可自定义的告警编辑器、报表编辑器（支持自定义SQL查询）；

5.B/S架构，通过浏览器异地轻松访问，支持Android、 Blackberry、iPhone手机终端访问；日志监控与分析需要使用C/S架构。

6.全球范围内在170多个国家超过110000名的SolarWinds用户；

7.财富500强中超过475位的SolarWinds的用户；

8.SolarWinds是NYSE上市公司；

三、SolarWinds功能优势

1.网络的监控与管理

- Network Performance Monitor（NPM）网络性能分析模块

- Network Configuration Managerment（NCM）网络配置管理模块

- NetFlow Traffic Analyzer（NTA）流量分析模块

- IP Address Manager（IPAM）IP地址管理模块

- IP SLA Manager（IPSLA）链路服务质量监控模块

- User Device Tracker（UDT）终端用户追踪模块

- Failover Engine（FOE）热备模块

- Enterprise Operations Console（EOC）企业级控制台

- Polling Engine （PE）扩展的轮询引擎模块

2.应用的监控与管理

- Application Performance Monitor （APM）应用性能监控模块

- Synthetic End User Monitor（SEUM）综合终端用户体验监控器

3.日志的监控与管理

- Log & Event Manager （LEM）日志与事件管理器

4.存储与备份的监控管理

- Storage Monitor （STM）存储监控器

- Backup Profiler（Back-up module）备份监控模块

5.虚拟化的监控与管理

- Virtualization Manager （VM）应用性能监控模块
